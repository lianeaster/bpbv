# Підтримка сайту кафедри БПБВ — інструкція (версія 1.0)

Цей документ для тих, хто підтримує сайт, але мало працював з HTML та JavaScript. Тут — тільки необхідні кроки без зайвої техніки.

---

## Що таке версія 1.0

Сайт вийшов у **версію 1.0**: основний контент, меню, мови, новини, галерея та мобільна версія готові. Далі потрібно лише оновлювати тексти, фото та новини за цим посібником.

---

## Структура проєкту (що для чого)

| Файл або папка | Призначення |
|----------------|-------------|
| **index.html** | Головна сторінка: розмітка секцій, посилання в меню, блоки з текстом і зображеннями. **Тексти на сторінці краще змінювати через translations.js** (див. нижче). |
| **style.css** | Всі візуальні стилі: кольори, шрифти, відступи, вигляд на телефоні. Якщо не впевнені — краще не чіпати. |
| **script.js** | Логіка сайту: меню, переклади, новини, анімації. Змінювати лише за потреби або з тим, хто знає JS. |
| **translations.js** | **Тут змінюють усі тексти сайту** для 4 мов: українська (uk), англійська (en), німецька (de), французька (fr). |
| **news.json** | **Тут додають і редагують новини** у блоці «Новини». Простий список записів. |
| **images/** | Усі зображення. Підпапки: **gallery** (галерея «життя кафедри»), **science** (наука, картка «Мікробний синтез»), **home** (головна/про кафедру), **faculty**, **clubs**, **international** тощо. |
| **assets/presentations/** | PDF-презентації кафедри для завантаження. |

Якщо не знаєте, у якому файлі щось змінити — спочатку перевірте **translations.js** і **news.json**.

## Як змінити тексти на сайті (без HTML)

Практично весь видимий текст беруться з **translations.js**. У файлі є блоки для кожної мови: `uk`, `en`, `de`, `fr`.

### Кроки

1. Відкрийте файл **translations.js** у будь-якому текстовому редакторі (Блокнот, Notepad++, VS Code тощо).
2. Знайдіть потрібний текст за ключем (наприклад, `'nav.about': 'Про кафедру'`). Можна користуватися пошуком (Ctrl+F) за словом українською, наприклад «Про кафедру» або «Історія».
3. Змініть **тільки те, що в лапках справа**. Приклад:
   - Було: `'nav.about': 'Про кафедру'`
   - Стало: `'nav.about': 'Про нас'`
4. **Однаковий ключ** потрібно змінити в усіх чотирьох мовах (uk, en, de, fr), інакше на інших мовах залишиться старий текст.
5. Збережіть файл. Після оновлення сторінки в браузері (F5 або перезавантаження) зміни з’являться.

### Приклад структури в translations.js

```text
uk: {
  'nav.about': 'Про кафедру',
  'hero.title1': 'Кафедра',
  ...
},
en: {
  'nav.about': 'About the department',
  'hero.title1': 'Department',
  ...
},
```

Не видаляйте лапки, коми та ключі (наприклад `'nav.about'`). Міняйте лише текст після двокрапки.

## Як додати або змінити новини

Новини зберігаються у файлі **news.json**.

### Формат одного запису

```json
{
  "title": "Заголовок новини",
  "date": "2024-11-17",
  "message": "Текст новини одним або кількома реченнями.",
  "url": "https://bpbv.nuft.edu.ua/"
}
```

- **title** — заголовок, який показується на сайті.
- **date** — дата у форматі РРРР-ММ-ДД (наприклад 2024-11-17).
- **message** — основний текст новини.
- **url** — посилання (наприклад на сторінку кафедри або детальну новину).

### Додати нову новину

1. Відкрийте **news.json**.
2. Знайдіть останній запис (блок у фігурних дужках `{ ... }`).
3. Після закриваючої дужки `}` поставте **кому** `,`, потім з нового рядка вставте новий блок у тому ж форматі.
4. Перевірте, що перед останнім записом є кома, а після останнього запису коми немає. У кінці файлу має бути `]`.
5. Збережіть файл.

### Редагувати існуючу новину

Змініть потрібні поля (`title`, `date`, `message`, `url`) всередині відповідного блоку `{ ... }`. Збережіть файл.

**Важливо:** у JSON не можна забувати коми між полями та записами і не можна ставити кому після останнього елемента в списку. Інакше сторінка новин може перестати відображатися.

## Як замінити фото

Усі зображення лежать у папці **images** та її підпапках.

| Що змінити | Папка | Як |
|------------|--------|-----|
| Фото в галереї «Яскраві моменти життя кафедри» | **images/gallery/** | Замініть файли **gallery-01.jpg** … **gallery-18.jpg** (один — **gallery-12.png**). Імена файлів краще не змінювати — тоді нічого не треба міняти в index.html. |
| Фото в блоці «Мікробний синтез та біоінженерія» | **images/science/** | Замініть файл **sci-10.jpg** на нове фото з тим самим ім’ям. |
| Фото в блоці «Про кафедру» (мозаїка) | **images/home/** | Використовуються **home-about-first.jpg**, **home-about-last.jpg** та інші home-*.jpg. Замініть файл з тим самим ім’ям. |
| Фото викладачів | **images/faculty/** | Замініть відповідний файл (наприклад faculty-01.jpg), зберігши ім’я файлу. |
| Фото гуртків | **images/clubs/** | Відповідні підпапки (brewers-club, wine-club, enogoly-club). Замініть файли, зберігши імена. |

**Порада:** якщо додаєте нове фото — збережіть його під тим самим ім’ям, що й старе (наприклад sci-10.jpg), щоб не потрібно було змінювати код.

## Як відкрити сайт на комп’ютері або в мережі

1. Відкрийте термінал (командний рядок) у папці проєкту (там, де лежать index.html, style.css тощо).
2. Виконайте команду:
   ```bash
   npm start
   ```
   Якщо питає встановлення пакетів — підтвердіть (Enter або Y).
3. У браузері відкрийте: **http://localhost:3000**
4. Щоб відкрити сайт з телефону в тій самій Wi‑Fi мережі: дізнайтеся IP комп’ютера (наприклад 192.168.1.184) і на телефоні відкрийте **http://192.168.1.184:3000**
5. Зупинити сервер: у терміналі натисніть **Ctrl+C**.

---

## Що краще не робити без досвіду

- **Не змінюйте структуру index.html** (не видаляйте блоки секцій, не перейменовуйте класи та id), якщо не розумієте HTML — можуть зламатися меню, переклади та стилі.
- **Не видаляйте і не перейменовуйте ключі** у **translations.js** (наприклад `'nav.about'`) — тільки текст після двокрапки.
- **Не чіпайте style.css і script.js**, якщо не потрібно змінювати вигляд або поведінку сайту.
- Перед масовими змінами **зробіть копію папки проєкту** або використовуйте Git (якщо вмієте).
